## 1. 这个天地怎么用(核心引擎)

**信念:不从地基先啃。地基的书是弹药库,不是入学考试。** 照常从真实创作切入,撞墙再下钻取对应弹药。

**核心循环(刻意练习)——学习发生在复盘里,不在出图里:**

> 写预测 → 只改一个变量 → 生成 → 对比 → 写复盘 → **顺手一句大白话喂进博客素材库**

最后一步不是额外任务,是把已经在做的动作多花 30 秒顺手记一下——这一格是 V2 加的,原因见第 6 节末尾。

**两种模式,分开,别让赶活吞掉学习:**

- **实验室模式**:慢、单变量、写预测和复盘。每周锁 3–4 个任务进实验室。
- **生产模式**:快、用已会的、不复盘。其余的活走这里。

**reading-debt(下钻欠债本)**:撞到"我不懂光/不懂构图/不懂模型为什么这样"的缺口 → 记进 `00-notes/reading-debt.md` → 攒够了去翻对应资源 → 读完回填一句大白话。

**swipe file(对比留痕本,V2.1 补上路径)**:roadmap 方法论里反复提到"存一份 swipe file,两周后就是你自己的私房参考手册",但之前一直没给它一个具体文件名——存进 `00-notes/swipe-file.md`,每条记录格式:日期 + 提示词 + 工具&版本 + steps/cfg/seed + 前后图路径。这份和 reading-debt 是两本不同的账:reading-debt 记"我不懂什么",swipe file 记"这个设置产出了这个结果",别混在一起写。

**记账触发点,两种都算,别只记第一种**:
1. 日常操作里撞到的"我不懂为什么"的瞬间(原有的)
2. 每篇 stage 文档结尾的 self-test,**答不上来的题**(V2 新增)——这些题是专门设计出来检验"是不是真懂"的,答不上来就是最精准的缺口信号,比日常模糊的"感觉不太懂"更值得记账。

## 2. 两条轨(你的真实创作,8:2)

- **主轨 80% · 电商写实(图生图 / 编辑)**:锁产品本体,只改背景/光/场景。
- **副轨 20% · 博主创作(从零生成)**:角色/场景/世界观/短视频/文案。
- **每两周一次交叉训练**:同一个 brief 换个工具再做一遍,观察不同工具对同一句话的反应——这本身就是在学机制。

**两栋楼并行爬:技能 + 品味。** 铁律:**出师才上楼;越往上越靠文档而非书。**

## 3.5 当前范围声明(V2 新增 —— 现在做什么,明确暂缓什么)

这一节存在的理由:半年后回头看,你需要能立刻分清"这条线没动是因为设计上还没轮到它",还是"这条线就是被忘了"。两者处理方式完全不同,不能混在一起。

**分两类,不是一类:**

### (a) 顺序性暂缓 —— 合理,不用管,轮到了自然会启动

技术阶梯本身是有序的。目前全部执行力(roadmap + 六篇 stage 详解)精确覆盖的是**技术阶梯②③**,也就是"图像编辑 + 精确控制"这一段。以下两项**不是遗漏,是还没轮到**:

- **技术阶梯④ 视频与动态**:触发条件 = 技术阶梯③正式毕业。到时候视觉 AIGC 全景图里的「四把尺子」和「当下主力 vs 上一代」两张表,就是你重新盘点视频生态该学哪个模型的起点——不用现在就去看。
- **四阶段产出 #2「自训 LoRA」**:注意,现有 roadmap 里 Stage 2 提到的 Lightning LoRA 是**用现成的**,不是**训练自己的**。真正的 LoRA 训练目前完全没有执行文档覆盖,这是合理的,因为它属于技术阶梯③的深水区,排在图像编辑打熟之后。触发条件同上:③毕业后,单开一份 LoRA 训练专项文档(不塞进现有 roadmap,避免把它撑得太杂)。
- **Flux Kontext(V2.1 新增声明)**:README 和全景图都把 Qwen-Image-Edit / Flux Kontext 并列为编辑层"真功夫"的两个选项,但 roadmap + 六篇 stage 详解目前从头到尾只覆盖 Qwen,Flux Kontext 一次没出现过——这条之前是隐性缺口,不是明说的排期。V2.1 给它一个默认值,不是替你做决定:**默认先把 Qwen-Image-Edit 单线走完到技术阶梯③毕业,再评估要不要/何时把 Flux Kontext 当第二个编辑模型引入**,理由是同时学两个编辑模型容易让"这次结果差是提示词问题还是模型选错了"的对比失真,违反单变量纪律。如果你其实想两个并行练,这条默认值需要你明确改掉,而不是让它继续隐性存在。

### (b) 并行轨道 —— V2.1 已补上执行计划

README 原本的设计里,审美阶梯和副业 20% 是**跟技术阶梯并行**的,不是排在后面。V2 发现这两条当时没有落地执行计划,V2.1 已经补齐:

- **审美阶梯** → 见 `aesthetics-ladder-roadmap.md`。五层各自对应哪本书、每天怎么练、什么时候算出师,都在里面。注意里面标出了一个重要的顺序调整:③色彩与光影因为直接反哺电商打光,建议插队,不必等①②读完。
- **副业 20%** → 见 `side-track-roadmap.md`。里面把"素材库攒着"拆成了四个阶段,并且明确了一条容易搞反的原则:**副业不需要等主业本地能力成熟**,该用托管工具(MJ/Kling/Runway 等)就直接用,因为副业练的是广度和产出,不是主业那种控制力。

这两份文档都跟主 roadmap 一样,只到"阶段+方法论+出师标准"这一层,不写逐日任务——同样的边界。

## 6. 几条护栏

- **毕业模型(治焦虑)**:会毕业的是基本功(光/镜头/构图/色彩 + 扩散原理),学透终身吃利息;不会毕业的是前沿(具体工具/版本)。精力大头压在会毕业那层,前沿从"重学"降级成"打补丁"。
- **评估纪律**:生成前先把成功标准**写死**(可判定,不是"好看")+ 给一个参照物对着比 + 定期找外部眼睛挑刺。
- **外部眼睛,具体化(V2 新增)**:不要让这条停在"定期"这种没有触发点的措辞上。**每个技术阶梯毕业检查通过后**,先别急着勾选——把毕业交付物发给至少一个外部人(社群/朋友/同行都行)挑刺一次,根据反馈决定是补一轮还是正式勾选。这样它绑在一个你本来就会做的动作上,不容易被跳过。
- **复现纪律**:每张图/每条片记全 —— **完整提示词 + 工具&版本 + seed + 参数 + 日期**。否则分不清变化是"你改了"还是"模型偷偷升级了"。
- **复盘与重访**:每个 drill 写复盘;**每月**抽一个旧 drill 冷启重做,查记忆衰减。
- **防过度求新**:不为"新"换工具。撞到现工具解决不了的真问题,才引入新模型。
- **脆弱依赖意识(V2 新增)**:任何一个具体工具/节点包/模型文件,只要你的管线依赖它,就有可能停止维护或悄悄断供(比如某个 ComfyUI 节点包原作者不再更新)。这类信息不要让它随手飘走——记进 `00-notes/fragile-dependencies.md`,配合技术阶梯⑤/roadmap 里"持续追新"的季度复查节奏一起处理,不用另开一条战线。
- **learning in public(你是博主,三位一体)**:把每个机制写成大白话发出去 —— 既是"真懂"测试、又是记忆锚、又是博主素材。**V2 备注**:这条不需要单独腾时间做——第 1 节核心循环末尾已经把"喂素材库"焊进日常动作里了,这里只负责定期把攒够的素材真正整理发出去。

## 8. 当前坐标 —— 两套系统对照表(V2 重写)

V1 里技术阶梯的 5 层打勾表,和 `comfyui-mastery-roadmap.md` 的 7 个 stage 是两套独立编号,勾选时容易把"roadmap 某个 stage 做完"直接当成"技术阶梯某层毕业",但两者不是一一对应的。下表把对应关系钉死:

| 技术阶梯层 | 对应 roadmap 阶段 | 毕业需要 | 状态 |
|---|---|---|---|
| ① 地基 | roadmap **Stage 1**(操作直觉部分) **+** Simon Prince 书对应章节(理论部分) | **两项都完成**才算数,不是任选一项 | 操作 [ ]　理论 [ ] |

| ③ 控制与一致性 | roadmap **Stage 3**(+ Stage 4 生产管线可视作③的巩固) | Stage 3 毕业检查通过 | [ ] |
